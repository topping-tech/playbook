وضعیت کلی فعلی

ساختار مرکزی هولدینگ ساخته شده و مسیر اصلی آن این است:

/www/holding-barzin

سرویس‌های اصلی CRM، NFC، Gozaresh و Veloxa فعال هستند. Gozaresh با systemd اجرا می‌شود و بقیه عمدتاً با PM2.

قانون امنیتی مهم

پسوردها، توکن‌ها، API Keyها و Connection Stringهای کامل نباید داخل فایل HTML عمومی نوشته شوند. فقط مسیر فایل‌های .env ثبت شده تا خود مدیر سرور بتواند آن‌ها را ببیند.

دستورات مهم مدیریت

بررسی سرویس‌های PM2

pm2 list
pm2 show nfc-api
pm2 logs nfc-api --lines 50

بررسی Gozaresh

sudo systemctl status gozaresh-api.service --no-pager -l
sudo systemctl status gozaresh-bot.service --no-pager -l
curl http://127.0.0.1:8000/health

بررسی PostgreSQL

sudo -u postgres psql -c "\l"
sudo -u postgres psql -c "\du"

بکاپ دیتابیس‌ها

پوشه بکاپ با chattr +i قفل می‌شود. برای گرفتن بکاپ جدید، اول قفل را بردار، بکاپ بگیر، سپس دوباره قفل کن.

sudo chattr -i /www/holding-barzin/backups
# run backup script here
sudo chattr +i /www/holding-barzin/backups

مسیر فعلی بکاپ‌ها:

/www/holding-barzin/backups

یادداشت بازیابی داده‌ها

دیتای تاریخی Gozaresh در دیتابیس فعلی پیدا نشد: reports=0، weekly_reports=0، users=1. سیستم از الان آماده ذخیره گزارش‌های جدید است.

برای NFC، بکاپ موجود قدیمی‌تر شامل ۳ پروفایل و ۲۰ کارت Restore شده است. دیتای جدیدتر روی VPS پیدا نشد و احتمالاً قبلاً روی Render یا دیتابیس خارجی بوده است.

چک‌لیست روزانه مدیر سرور

  • بررسی pm2 list
  • بررسی وضعیت gozaresh-api.service و gozaresh-bot.service
  • اجرای بکاپ دیتابیس‌ها
  • نگهداری بکاپ خارج از سرور
  • حذف نکردن پوشه‌ها قبل از گرفتن بکاپ
